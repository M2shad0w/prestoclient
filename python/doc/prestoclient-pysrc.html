 31      PrestoClient 
 32      ============ 
 33   
 34      PrestoClient implements a Python class to communicate with a Presto server. 
 35      Presto (U{http://prestodb.io/}) is a fast query engine developed 
 36      by Facebook that runs distributed queries against a (cluster of) 
 37      Hadoop HDFS servers (U{http://hadoop.apache.org/}). 
 38      Presto uses SQL as its query language. Presto is an alternative for 
 39      Hadoop-Hive. 
 40   
 41      PrestoClient was developed and tested on Presto versions 0.52 to 0.68. Python version used is 2.7.6 
 42   
 43      You can use this class with this sample code: 
 44   
 45      >>> import prestoclient 
 46      >>> 
 47      >>> sql = "SHOW TABLES" 
 48      >>> 
 49      >>> # Replace localhost with ip address or dns name of the Presto server running the discovery service 
 50      >>> presto = prestoclient.PrestoClient("localhost") 
 51      >>> 
 52      >>> if not presto.runquery(sql): 
 53      >>>     print "Error: ", presto.getlasterrormessage() 
 54      >>> else: 
 55      >>>     # We're done now, so let's show the results 
 56      >>>     print "Columns: ", presto.getcolumns() 
 57      >>>     if presto.getdata(): print "Datalength: ", presto.getnumberofdatarows(), " Data: ", presto.getdata() 
 58   
 59      Presto client protocol 
 60      ====================== 
 61   
 62      The communication protocol used between Presto clients and servers is not documented yet. It seems to 
 63      be as follows: 
 64   
 65      Client sends http POST request to the Presto server, page: "/v1/statement". Header information should 
 66      include: X-Presto-Catalog, X-Presto-Source, X-Presto-Schema, User-Agent, X-Presto-User. The body of the 
 67      request should contain the sql statement. The server responds by returning JSON data (http status-code 200). 
 68      This reply may contain up to 3 uri's. One giving the link to get more information about the query execution 
 69      ('infoUri'), another giving the link to fetch the next packet of data ('nextUri') and one with the uri to 
 70      cancel the query ('partialCancelUri'). 
 71   
 72      The client should send GET requests to the server (Header: X-Presto-Source, User-Agent, X-Presto-User. 
 73      Body: empty) following the 'nextUri' link from the previous response until the servers response does not 
 74      contain an 'nextUri' link anymore. When there is no 'nextUri' the query is finished. If the last response 
 75      from the server included an error section ('error') the query failed, otherwise the query succeeded. If 
 76      the http status of the server response is anything other than 200 with Content-Type application/json, the 
 77      query should also be considered failed. A 503 http response means that the server is (too) busy. Retry the 
 78      request after waiting at least 50ms. 
 79      The server response may contain a 'state' variable. This is for informational purposes only (may be subject 
 80      to change in future implementations). 
 81      Each response by the server to a 'nextUri' may contain information about the columns returned by the query 
 82      and all- or part of the querydata. If the response contains a data section the columns section will always 
 83      be available. 
 84   
 85      The server reponse may contain a variable with the uri to cancel the query ('partialCancelUri'). The client 
 86      may issue a DELETE request to the server using this link. Response http status-code is 204. 
 87   
 88      The Presto server will retain information about finished queries for 15 minutes. When a client does not 
 89      respond to the server (by following the 'nextUri' links) the server will cancel these 'dead' queries after 
 90      5 minutes. These timeouts are hardcoded in the Presto server source code. 
 91
